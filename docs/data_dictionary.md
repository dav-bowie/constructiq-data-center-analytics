# Data Dictionary — ConstructIQ Star Schema

All tables live in Amazon Redshift Serverless, schema: `public`.
Generated by the AWS Glue ETL job (`glue-etl/constructiq_etl.py`).

---

## Fact Tables

### `fact_project_cost`
One row per cost code per building. Source: Procore cost ledger export.

| Column | Type | Description |
|---|---|---|
| `cost_key` | INTEGER | Surrogate key from source system |
| `building_id` | VARCHAR | Building identifier (BLD-A through BLD-D) |
| `cost_code` | VARCHAR | Procore cost code (e.g. `01.10.001`) |
| `wbs_level_1` | VARCHAR | Top-level WBS category (e.g. `01-SITEWORK`) |
| `wbs_level_2` | VARCHAR | Second-level WBS category (e.g. `01.10-Earthwork`) |
| `vendor_name` | VARCHAR | Standardized vendor name (fuzzy-matched to canonical list) |
| `contract_value` | BIGINT | Original contract value in USD |
| `approved_budget` | BIGINT | Approved budget at completion (BAC) in USD |
| `committed_cost` | BIGINT | Total committed cost including approved change orders |
| `actual_cost_to_date` | BIGINT | Actual cost incurred to snapshot date (AC) |
| `forecasted_final_cost` | BIGINT | Projected total cost at completion |
| `variance` | BIGINT | `approved_budget - forecasted_final_cost`. Negative = over budget |
| `earned_value` | BIGINT | Budgeted cost of work physically completed (EV) |
| `planned_value` | BIGINT | Budgeted cost of work scheduled by snapshot date (PV) |
| `percent_complete_actual` | DOUBLE | Physical percent complete (0.0 to 1.0) |
| `percent_complete_baseline` | DOUBLE | Baseline scheduled percent complete (0.0 to 1.0) |
| `cpi` | DOUBLE | Cost Performance Index = EV / AC. Null if AC = 0 |
| `spi` | DOUBLE | Schedule Performance Index = EV / PV. Null if PV = 0 |
| `eac` | DOUBLE | Estimate at Completion = Budget / CPI. Null if CPI = 0 |
| `vac` | DOUBLE | Variance at Completion = Budget - EAC. Null if EAC = null |
| `cost_variance_pct` | DOUBLE | `(Budget - Forecast) / Budget × 100`. Negative = over budget |

### `fact_change_orders`
One row per change order. Source: Change order log export.

| Column | Type | Description |
|---|---|---|
| `change_order_id` | VARCHAR | Unique CO identifier (e.g. `CO-1001`) |
| `building_id` | VARCHAR | Building the CO applies to |
| `co_type` | VARCHAR | CO type: `OCO` (Owner-directed), `COR` (Contractor request), `PCO` (Proposed) |
| `description` | VARCHAR | Plain-English description of the scope change |
| `submitted_date` | DATE | Date contractor submitted the CO (normalized to YYYY-MM-DD) |
| `approved_date` | DATE | Date CO was approved. `9999-12-31` if still pending/under review |
| `status` | VARCHAR | `Approved`, `Rejected`, `Pending`, `Under Review` |
| `cost_impact` | DOUBLE | Cost impact in USD. Positive = cost increase |
| `schedule_impact_days` | INTEGER | Schedule impact in calendar days |
| `co_approval_lag_days` | INTEGER | Days from submission to approval. Null for pending COs |
| `responsible_party` | VARCHAR | Party responsible for the CO (Owner, Contractor, etc.) |
| `wbs_code` | VARCHAR | WBS code the CO is charged to |
| `wbs_level_1` | VARCHAR | Top-level WBS category |

---

### `fact_schedule`
One row per schedule activity. Source: Primavera P6 export.

| Column | Type | Description |
|---|---|---|
| `activity_id` | VARCHAR | Unique activity identifier (e.g. `ACT-0001`) |
| `building_id` | VARCHAR | Building the activity belongs to |
| `activity_name` | VARCHAR | Activity description (e.g. `Mass Excavation`) |
| `wbs_level_1` | VARCHAR | Top-level WBS category |
| `wbs_code` | VARCHAR | WBS code |
| `planned_start` | DATE | Baseline planned start date |
| `planned_finish` | DATE | Baseline planned finish date |
| `actual_start` | DATE | Actual start date. Null if activity not yet started |
| `actual_finish` | DATE | Actual finish date. Null if activity not yet complete |
| `baseline_duration` | INTEGER | Planned duration in calendar days |
| `actual_duration` | INTEGER | Actual duration in calendar days. Null if in progress |
| `percent_complete` | DOUBLE | Physical percent complete (0.0 to 1.0) |
| `total_float_days` | INTEGER | Schedule float in days. Negative = behind critical path |
| `critical_path_flag` | VARCHAR | `Y` if activity is on the critical path, `N` otherwise |

---

### `fact_pay_applications`
One row per contractor pay application per period. Source: Pay application register.

| Column | Type | Description |
|---|---|---|
| `pay_app_id` | VARCHAR | Unique pay app identifier (e.g. `PA-5001`) |
| `building_id` | VARCHAR | Building the pay app applies to |
| `period` | VARCHAR | Billing period (e.g. `2023-06`) |
| `period_sequence` | INTEGER | Sequential period number (1 = first billing period) |
| `contractor_name` | VARCHAR | Standardized contractor name (fuzzy-matched) |
| `trade` | VARCHAR | Trade category (e.g. `GC`, `Electrical`, `Mechanical`) |
| `scheduled_value` | BIGINT | Total contract value (denominator for percent billed) |
| `work_completed_this_period` | BIGINT | Work billed in this period only (not cumulative) |
| `work_completed_to_date` | BIGINT | Cumulative work billed through this period |
| `stored_materials` | BIGINT | Value of materials stored on site but not yet installed |
| `total_earned` | BIGINT | `work_completed_to_date + stored_materials` |
| `retainage_pct` | DOUBLE | Retainage percentage withheld (typically 10.0) |
| `retainage_held` | BIGINT | Total retainage withheld to date in USD |
| `net_payment_due` | BIGINT | Amount payable this period after retainage |
| `pay_app_status` | VARCHAR | `Certified`, `Submitted`, `Under Review`, `Rejected` |

## Dimension Tables

### `dim_building`
One row per building. Provides static program metadata.

| Column | Type | Description |
|---|---|---|
| `building_id` | VARCHAR | Primary key (BLD-A, BLD-B, BLD-C, BLD-D) |
| `campus` | VARCHAR | Campus name (`Northern Virginia Data Center Campus`) |
| `client` | VARCHAR | Client name (`Amazon Web Services`) |
| `program_value_usd` | BIGINT | Total program value in USD ($2,100,000,000) |
| `snapshot_date` | DATE | Reporting snapshot date (2025-09-30) |

---

### `dim_wbs`
One row per WBS code. Provides the Work Breakdown Structure hierarchy.

| Column | Type | Description |
|---|---|---|
| `wbs_code` | VARCHAR | WBS or cost code (e.g. `01.10.001`) |
| `wbs_level_1` | VARCHAR | Top-level category (e.g. `01-SITEWORK`, `02-STRUCTURE`) |
| `wbs_level_2` | VARCHAR | Sub-category (e.g. `01.10-Earthwork`). Null for some codes |

---

### `dim_vendor`
One row per standardized vendor/contractor. Deduplicated via fuzzy matching in ETL.

| Column | Type | Description |
|---|---|---|
| `vendor_id` | INTEGER | Surrogate key (sequential, generated by ETL) |
| `vendor_name` | VARCHAR | Canonical vendor name after fuzzy standardization |
| `trade` | VARCHAR | Trade classification from pay applications. Null if vendor is cost-only |

---

### `dim_date`
One row per calendar day, 2023-01-01 through 2026-12-31. Standard date spine.

| Column | Type | Description |
|---|---|---|
| `full_date` | DATE | Primary key — the calendar date |
| `date_key` | INTEGER | Integer surrogate key in YYYYMMDD format (e.g. 20250930) |
| `year` | INTEGER | Calendar year (2023–2026) |
| `quarter` | INTEGER | Calendar quarter (1–4) |
| `month` | INTEGER | Month number (1–12) |
| `month_name` | VARCHAR | Month name (e.g. `September`) |
| `week_of_year` | INTEGER | ISO week number (1–53) |
| `day_of_week` | INTEGER | Day of week: 1 = Sunday, 7 = Saturday |
| `day_name` | VARCHAR | Day name (e.g. `Tuesday`) |
| `is_weekend` | BOOLEAN | True if Saturday or Sunday |
| `is_month_end` | BOOLEAN | True if the last day of the month |
| `fiscal_year` | INTEGER | Fiscal year (same as calendar year in this program) |
| `fiscal_quarter` | INTEGER | Fiscal quarter (same as calendar quarter in this program) |
